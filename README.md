# 시각 장애인을 위한 음성 기반 길 안내 앱 **NaviA**

## 📌 프로젝트 소개
**NaviA**는 시각 장애인이 스마트폰 화면을 보지 않고도 목적지를 설정하고 안전하게 보행할 수 있도록 돕는 안드로이드 애플리케이션입니다. 
복잡한 화면 터치 대신 **전체 화면 롱프레스(Long Press)** 제스처와 **Push-to-Talk** 방식의 음성 인식을 핵심 인터페이스로 채택하여 접근성을 극대화했습니다.

## ✨ 주요 기능

### 1. 직관적인 제스처 & 음성 인터페이스 (Push-to-Talk)
- **화면 길게 누르기 (Long Press):** 화면의 어느 곳이든 길게 누르면 "듣고 있어요!"라는 피드백과 함께 음성 인식이 시작됩니다.
- **손 떼면 인식 종료 (Release to Stop):** 말을 마친 후 화면에서 손을 떼면 즉시 인식을 종료하고 명령을 처리합니다.
- **음성 안내 (TTS):** 현재 상태, 인식 결과, 오류 상황 등을 음성으로 상세하게 안내합니다.

### 2. 다감각 피드백 시스템 (Haptic & Sound)
- **햅틱 피드백:** 터치(틱), 길게 누르기(더블 진동), 오류(긴 진동) 등 상태에 따라 명확히 구분된 진동 패턴을 제공합니다.
- **사운드 피드백:** 음성 인식 시작음, 성공/실패 알림음을 통해 시각 정보 없이도 앱의 상태를 파악할 수 잇습니다.

### 3. 고대비(High Contrast) & 접근성 UI
- **고대비 테마:** 검은색 배경에 노란색(Yellow), 흰색(White), 녹색(Green) 등 시인성이 높은 색상을 사용하여 저시력 사용자도 쉽게 식별할 수 있습니다.
- **가독성 높은 글꼴:** `sans-serif-medium` 및 `sans-serif-black` 폰트를 적용하여 글자의 식별력을 높였습니다.
- **직관적인 아이콘:** 걷는 사람 픽토그램 등 의미 전달이 명확한 아이콘을 사용했습니다.

### 4. 핵심 기능 모드
1.  **메인(Main):** 앱 대기 상태. 음성 명령 대기.
2.  **네비게이션(Navigation):** 목적지 설정 및 경로 안내 ("네비", "길안내" 명령).
3.  **보행 모드(Walking):** 보행 중 안전 정보 및 걸음 수 안내 ("보행", "걷기", "산책" 명령).

## 🛠 기술 스펙
- **Language:** Kotlin
- **Architecture:** MVVM, Single Activity
- **Accessibility APIs:** 
  - `SpeechRecognizer` (Android Speech API)
  - `TextToSpeech` (TTS)
  - `VibratorManager` (Haptic Feedback)

## 🚀 설치 및 실행 방법
1.  Android Studio에서 `NaviA` 프로젝트를 엽니다.
2.  실제 안드로이드 기기 또는 에뮬레이터에 빌드합니다. (음성 인식 테스트를 위해 실제 기기 권장)
3.  앱 최초 실행 시 **마이크(Audio)** 및 **위치(Location)** 권한을 허용해야 정상 작동합니다.
